Anthropic MCP (mcp-server-git): prompt injection → file read/delete & potential code execution (CVE-2025-68143/44/45)

2026-01-30 • Category: AI CVEs

  • What happened: Cyata disclosed three vulnerabilities in mcp-server-git (Anthropic’s official / reference Git server for MCP) that can be triggered via prompt injection — i.e., by influencing what the assistant reads (README, issue text, web page).
  • Why this is different: the issues were described as working “out of the box” in default installs, not just in unusual or obviously unsafe configurations.
  • Core design lesson: once an LLM can call tools, “prompt injection” stops being a content problem and becomes an execution surface (tool calls + attacker-controlled arguments).
  • CVE-2025-68143 (unrestricted git_init): repository initialization could be directed at attacker-chosen paths, enabling destructive or enabling steps in larger chains.
  • CVE-2025-68145 (path validation bypass): tool calls could operate on directories beyond the configured repository, letting an attacker point Git operations at unintended locations.
  • CVE-2025-68144 (argument injection in git_diff): unsanitized arguments could allow file overwrite via Git flags (e.g., --output), effectively enabling arbitrary file clobbering / deletion patterns.
  • Impact amplification: when mcp-server-git runs alongside a filesystem-writing capability (filesystem MCP server or IDE-native file writes), Cyata demonstrated a path to code execution (via Git filters) without needing execute bits on files.
  • Fix status: Anthropic accepted the report in Sep 2025 and shipped fixes in Dec 2025; Cyata recommends upgrading to mcp-server-git 2025.12.18+.

Why it matters

  • MCP is becoming “invisible plumbing” for agentic workflows (IDE assistants, internal copilots, automation). Reference implementations tend to get copy/pasted into production.
  • Tool composition creates emergent risk: Git access alone might “just” leak context; Git + file write + prompt injection can become real system compromise.
  • Security reviews must treat tool servers like daemons: input validation, path allowlists, least privilege, and network exposure rules matter as much as “model safety.”

What to do

  1. Upgrade immediately: update mcp-server-git to 2025.12.18+ (or newer) everywhere you run MCP.
  2. Stop running “Git + filesystem write” by default: only enable filesystem-writing tools for sessions/users that truly need it; prefer read-only modes where possible.
  3. Enforce path allowlists: MCP servers should only operate inside a specific workspace root; reject absolute paths and .. traversal even if the tool “could handle it.”
  4. Constrain execution environment: run MCP servers as low-privilege users, inside sandboxes/containers, with restrictive file permissions and minimal environment secrets.
  5. Monitor for weird Git artifacts: unexpected .git/ directories created outside repos, sudden changes to .git/config, .gitattributes, and new filter definitions are high-signal.
