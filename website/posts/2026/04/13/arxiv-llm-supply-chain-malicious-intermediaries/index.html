arXiv:2604.08407 — Malicious Intermediary Attacks on LLM Supply Chain

AI relevance: This research exposes critical supply chain vulnerabilities in LLM agent ecosystems where API routers can intercept and manipulate tool-calling requests between agents and upstream providers.

Key Findings

  • 28 paid routers purchased from Taobao, Xianyu, and Shopify analyzed
  • 400 free routers collected from public communities examined
  • 1 paid and 8 free routers actively injecting malicious code
  • 2 routers deploying adaptive evasion triggers
  • 17 routers touching researcher-owned AWS canary credentials
  • 1 router draining ETH from researcher-owned private key
  • Leaked OpenAI key generated 100M GPT-5.4 tokens
  • Weakly configured decoys yielded 2B billed tokens
  • 401 sessions running in autonomous YOLO mode

Why It Matters

LLM agents increasingly rely on third-party API routers that operate as application-layer proxies with full plaintext access to JSON payloads, yet no provider enforces cryptographic integrity between client and upstream model. This creates a massive attack surface for supply chain attacks.

What To Do

  • Implement client-side defenses: fail-closed policy gates, response-side anomaly screening, append-only transparency logging
  • Demand cryptographic integrity guarantees from API providers
  • Audit third-party router implementations before deployment
  • Monitor token usage and credential exposure patterns
  • Consider self-hosted routing solutions for sensitive workloads
