arXiv — Environment-Injected Memory Poisoning Attacks on Web Agents

AI relevance: This research demonstrates how AI web agents with memory capabilities can be compromised through subtle environmental observations, creating persistent attack surfaces that span websites and sessions, fundamentally challenging the security model of personalized AI assistants.

  • eTAMP attack — Environment-injected Trajectory-based Agent Memory Poisoning enables cross-session, cross-site compromise
  • No direct memory access required — Contamination occurs through environmental observation alone
  • High success rates — Up to 32.5% on GPT-5-mini, 23.4% on GPT-5.2, and 19.5% on GPT-OSS-120B
  • Frustration Exploitation — Agents under environmental stress become dramatically more susceptible (ASR increases up to 8×)
  • Persistent compromise — Single poisoned observation can affect future tasks on different websites
  • Bypasses permission defenses — Attacks work even when agents have restricted memory access permissions

Why it matters

This research fundamentally shifts the threat model for AI agents with memory capabilities. Unlike traditional attacks requiring direct memory injection or shared memory access, eTAMP demonstrates that simply viewing manipulated content (like a product page) can permanently poison an agent's memory, creating a persistent backdoor that activates during future tasks on completely different websites.

What to do

  • Implement memory sanitization routines for AI agents
  • Develop detection mechanisms for memory poisoning patterns
  • Consider memory compartmentalization strategies
  • Audit agent memory usage and environmental interactions
  • Monitor for abnormal memory access patterns across sessions
