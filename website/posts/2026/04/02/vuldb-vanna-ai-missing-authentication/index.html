VulDB — vanna-ai Missing Authentication (CVE-2026-5320)

AI relevance: vanna-ai is an open-source AI-powered data analysis tool, and missing authentication in its API endpoints can expose sensitive data processed by AI systems to unauthorized access.

Critical Vulnerability Details

  • CVE: CVE-2026-5320
  • Component: vanna-ai Chat API Endpoint
  • Affected endpoint: /api/vanna/v2/
  • Vulnerability type: Missing authentication
  • CVSS severity: Critical
  • Affected versions: Up to 2.0.2

Why It Matters

vanna-ai is used for natural language data analysis and visualization. This missing authentication vulnerability allows:

  • Unauthorized access to chat API endpoints
  • Potential exposure of sensitive data analysis results
  • Manipulation of AI-generated data visualizations
  • Compromise of data integrity in AI-powered analytics workflows
  • Potential data exfiltration through unauthenticated API calls

What To Do

  • Immediately update to the latest patched version
  • Implement proper authentication mechanisms for all API endpoints
  • Add API key validation or OAuth protection
  • Enable proper access controls and rate limiting
  • Audit all API endpoints for similar authentication gaps
  • Monitor API access logs for unauthorized requests
