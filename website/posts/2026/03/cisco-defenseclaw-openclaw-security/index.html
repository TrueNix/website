Cisco Security Blog — DefenseClaw: Open-source AI agent security governance

AI relevance: DefenseClaw addresses critical security gaps in OpenClaw AI agent ecosystems by providing runtime security scanning, threat detection, and enforcement capabilities specifically designed for AI agent deployments.

Cisco has announced DefenseClaw, an open-source governance layer for OpenClaw agents that provides comprehensive security scanning and threat detection capabilities. The tool represents a significant advancement in securing AI agent ecosystems against emerging threats.

Key capabilities

  • Pre-execution scanning: Scans all skills, tools, plugins, and generated code before execution with five integrated security tools
  • Runtime threat detection: Monitors agent communications and detects threats during operation, not just at deployment
  • Enforcement capabilities: Block/allow lists with immediate enforcement (under 2 seconds) without requiring agent restarts
  • Built-in observability: Seamless integration with Splunk for comprehensive telemetry and monitoring
  • OpenShell integration: Works with NVIDIA's OpenShell sandboxing framework for enhanced security

Why it matters

The rapid adoption of OpenClaw has created significant security challenges, with over 135,000 exposed instances and multiple critical vulnerabilities (including CVE-2026-25253 for remote code execution). DefenseClaw provides the operational governance layer needed to secure AI agent deployments at scale.

Traditional security tools are insufficient for AI agent ecosystems because:

  • Agents have broad system access and network reach
  • Community-contributed skills create supply chain risks
  • Self-evolving capabilities require continuous monitoring
  • Prompt injection and data exfiltration threats are unique to AI systems

What to do

  • Implement DefenseClaw: Deploy the open-source tool for existing OpenClaw installations
  • Scan third-party skills: Use integrated scanners to vet community-contributed components
  • Establish block lists: Create and maintain lists of known malicious skills and MCP servers
  • Enable observability: Connect to Splunk or other monitoring systems for comprehensive visibility
  • Stay updated: Monitor Cisco's AI Defense blog for ongoing security updates and best practices
