XM Cyber — AWS Bedrock attack vectors: 8 ways to hijack AI agents

AI relevance: These attack vectors specifically target AI agent infrastructure in AWS Bedrock, demonstrating how attackers bypass AI security controls to compromise enterprise systems connected to AI workflows.

XM Cyber researchers identified eight validated attack vectors in AWS Bedrock that exploit the platform's connectivity to enterprise systems:

  • Model Invocation Log Attacks: Redirect logs to attacker-controlled S3 buckets or delete forensic evidence
  • Knowledge Base Data Source Attacks: Bypass models to pull raw data directly from S3/Salesforce/SharePoint
  • Knowledge Base Data Store Attacks: Steal credentials to gain administrative access to vector databases
  • Direct Agent Attacks: Rewrite agent prompts or attach malicious executors
  • Indirect Agent Attacks: Inject malicious code into Lambda functions agents depend on
  • Flow Attacks: Inject sidecar nodes into workflows or swap encryption keys
  • Guardrail Attacks: Weaken or remove content filtering and safety controls
  • Managed Prompt Attacks: Poison prompt templates across entire environments

Why it matters

AWS Bedrock's power comes from connecting AI models directly to enterprise data and systems, but this connectivity creates attack paths that bypass traditional security boundaries. A single over-privileged identity can compromise entire AI workflows, exfiltrate sensitive data, or gain lateral movement into critical infrastructure.

These attacks target the permissions and configurations surrounding AI models rather than the models themselves, making them difficult to detect with traditional application monitoring tools. The research demonstrates that AI security requires hardening the entire infrastructure stack, not just the AI components.

What to do

  • Conduct thorough permission audits of all Bedrock identities and attached policies
  • Implement least-privilege access for Bedrock components and integrated services
  • Monitor Bedrock API calls and configuration changes for suspicious activity
  • Segment AI workloads from critical production systems where possible
  • Regularly review and test guardrail configurations and prompt management systems
  • Implement strong encryption and key management for Bedrock workflows
