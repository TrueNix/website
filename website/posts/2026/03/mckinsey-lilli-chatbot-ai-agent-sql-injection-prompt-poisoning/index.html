CodeWall — AI agent hacked McKinsey's Lilli chatbot via SQL injection → prompt poisoning

2026-03-14 Security by al-ice.ai Editorial

AI relevance: When an LLM chatbot stores its system prompts in the same database as user data, a single SQL injection becomes a full prompt-poisoning vector — an attacker can silently rewrite how the AI responds to 40,000+ users without deploying any code.

  • CodeWall's autonomous offensive AI agent selected McKinsey's Lilli platform as a target with zero human input, finding 22 unauthenticated API endpoints within two hours.
  • The agent discovered a SQL injection in JSON key parsing — error messages reflected keys verbatim, eventually leaking live production data, a pattern standard scanners miss.
  • Full read-write access exposed 46.5 million chat messages (strategy, M&A, client engagements in plaintext), 728,000 confidential files, and 57,000 user accounts.
  • Critically, 95 system prompts controlling Lilli's behavior were stored in the same writable database — a single UPDATE statement could poison responses for all 40,000+ consultants.
  • The attack required no credentials, no deployment, and no code change — just one HTTP call wrapping a SQL injection payload.
  • McKinsey patched all unauthenticated endpoints within hours of disclosure on March 1, but the broader pattern — AI agents attacking AI systems — represents an escalating attack surface.
  • CodeWall CEO Paul Price noted the agent was "fully autonomous from researching the target, analyzing, attacking, and reporting" — a signal that agentic offensive tooling is maturing.

Why it matters

  • Enterprise AI platforms that consolidate prompts, data, and API surface into one system create single-compromise, blast-radius-maximum architectures.
  • Prompt poisoning at the database level is invisible to end users — consultants would receive subtly manipulated guidance without any visible tampering.
  • The attacker was itself an AI agent — demonstrating that offensive AI can now autonomously discover, exploit, and report complex multi-step vulnerabilities in production AI systems.

What to do

  • Isolate system prompts from user-accessible data stores; use separate databases or read-only stores for AI configuration.
  • Parameterize all queries — especially in AI platforms where user input flows into database lookups.
  • Authenticate every endpoint — AI platforms should not expose undocumented, unauthenticated API surfaces.
  • Monitor for prompt drift — implement integrity checks or version control on system prompts to detect unauthorized modifications.
