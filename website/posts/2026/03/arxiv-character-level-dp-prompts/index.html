arXiv:2603.26032 — Character-Level Differential Privacy for LLM Prompts

Published March 30, 2026

Security Research

AI relevance: This research addresses a critical security challenge in AI agent deployments where user prompts containing sensitive PII could be exposed to third-party LLM providers, making it essential for secure AI tooling and agent infrastructure.

Researchers from academia have proposed a novel approach to protect sensitive user prompts in LLM interactions using character-level differential privacy:

  • Uses randomized response mechanism to perturb each character independently
  • Remote LLM first performs prompt restoration before downstream tasks
  • Common words reconstruct easily while rare/sensitive words remain protected
  • Works without explicit PII identification - privacy by design approach
  • Tested on medical (i2b2/UTHealth) and email (Enron) datasets
  • Evaluated with Llama-3.1 8B Instruct and GPT-4o mini models
  • Shows better privacy-utility tradeoff compared to word-level DP
  • Outperforms rule-based PII redaction baselines

Why This Matters

As AI agents become more integrated into workflows handling sensitive data, prompt privacy becomes a critical security concern. Traditional PII redaction requires explicit identification of sensitive information, which can be error-prone. This character-level differential privacy approach provides a mathematical guarantee of privacy while maintaining utility for common language tasks.
