Code Is Cheap Now. Craft Is Priceless.

February 24, 2026 Catalyst
The Ladybird browser just ported 25,000 lines of JavaScript engine code from C++ to Rust in two weeks. Not a team. One person. With AI.

Let that sink in for a moment. A browser engine—one of the most complex pieces of software humans build—just underwent a fundamental language migration in the time it takes most teams to argue about naming conventions.

Zero regressions. Byte-for-byte identical output. 52,898 test262 tests passing. 12,461 regression tests green.

The Old Math Manual rewrite: multiple months of senior engineering time
The New Math AI-assisted translation: two weeks, hundreds of directed prompts

This isn't a story about AI replacing developers. It's a story about what happens when the cost structure of an entire industry inverts overnight.

The Inversion

For decades, we've operated under one immutable constraint: code is expensive. Every line costs time. Every feature requires justification. Every refactor demands a business case.

Our entire professional culture evolved around this scarcity:

  • Sprint planning exists because coding time is precious
  • Technical debt accumulates because cleanup "isn't worth it"
  • Documentation lags because writing takes too long
  • Edge cases go untested because coverage is expensive
  • Refactoring gets deferred because "if it works, don't touch it"

Every "should we build this?" conversation has been secretly asking: "Is this worth the time?"

Now the time is free. The question remains.

Simon Willison puts it plainly: "Writing code is cheap now." The challenge isn't typing characters into a file anymore. The challenge is knowing what to build, how to verify it works, and whether it solves the right problem.

What Ladybird Actually Did

Andreas Kling didn't just throw code at Claude and ship it. Read his post carefully:

"This was human-directed, not autonomous code generation. I decided what to port, in what order, and what the Rust code should look like. It was hundreds of small prompts, steering the agents where things needed to go."

Then came the adversarial review: multiple AI models analyzing the output for mistakes and anti-patterns. Then lockstep verification—running both C++ and Rust pipelines simultaneously on real web traffic, confirming identical behavior.

The AI didn't decide anything. It executed with superhuman speed what a human architect designed with decades of expertise.

What Got Cheaper

  • Typing code
  • Syntax translation
  • Boilerplate generation
  • Pattern replication
  • Mechanical refactoring

What Stayed Expensive

  • Architectural decisions
  • Correctness verification
  • Problem definition
  • Quality judgment
  • Long-term maintainability

The Burden Shifts

Here's what nobody tells you about cheap code: it makes bad decisions more expensive.

When writing code took a week, you had a week to realize you were building the wrong thing. Now you can build the wrong thing in an afternoon—and the cost of that mistake compounds faster than ever.

The bottleneck moved. It's no longer "how fast can we type?" It's:

  • Do we understand the problem? AI will happily solve the wrong problem at lightning speed.
  • Can we verify correctness? Ladybird had 65,000+ tests. Do you?
  • Will this be maintainable? Code that works today but breaks tomorrow is worse than no code.
  • Does this actually matter? Just because you can build it in an hour doesn't mean you should.
"Any time our instinct says 'don't build that, it's not worth the time' fire off a prompt anyway... where the worst that can happen is you check ten minutes later and find that it wasn't worth the tokens."
— Simon Willison

This is the new discipline: learning when to say yes to experiments that cost nothing but attention.

Craft in the Age of Abundance

The Ladybird port is remarkable not because AI wrote Rust code. It's remarkable because a human knew exactly what to ask for, how to verify it, and when to stop.

That's craft.

Craft is knowing that "byte-for-byte identical output" is the right success metric for a language migration. Craft is running adversarial reviews because you don't trust the first draft. Craft is building lockstep verification because correctness matters more than speed.

Craft is the thing AI makes more valuable, not less.

Code is cheap. Judgment is priceless. Taste is irreplaceable.

The New Game

We're entering an era where the limiting reagent isn't engineering capacity—it's clarity of thought.

Can you articulate what "good" looks like? Can you design a verification strategy? Can you recognize when generated code is subtly wrong? Can you maintain a coherent architecture across 100 AI-assisted changes?

These were always important skills. Now they're the only skills that matter.

The developers who thrive won't be the ones who can code fastest. They'll be the ones who can:

  • Architect with precision
  • Verify with rigor
  • Judge with taste
  • Decide with conviction

The rest? The AI will handle it. In two weeks instead of two months.

Your Move

So here's the uncomfortable question: What are you still doing manually because "it's not worth the time" that you should be delegating to an AI right now?

And the even more uncomfortable one: What are you building with AI that you shouldn't be building at all?

Code is cheap now. Your attention isn't. Your judgment isn't. Your taste isn't.

Spend them wisely.
