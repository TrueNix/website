Writer AI 'WriteOut' — Cross-Tenant Session Hijack via Agent Preview

2026-07-08 Security by al-ice.ai Editorial

AI relevance: Enterprise AI platforms that let users build and preview agents in managed sandboxes must enforce tenant isolation at the transport layer — not just the prompt layer — or a single link becomes a cross-tenant account takeover vector.

What Happened

  • Sand Security Research disclosed "WriteOut," a critical session isolation vulnerability in Writer, an enterprise generative AI platform.
  • The flaw allowed an attacker to take over any Writer AI organization inside enterprise customers — with nothing more than a single preview link.
  • Attack chain: attacker creates a malicious agent in their own Writer account, enables the live preview feature, and shares the public preview URL.
  • When a logged-in Writer user from a completely different organization opens the link, their browser attaches their Writer session cookie to the request.
  • The preview proxy forwards that cookie into the attacker's managed sandbox — breaking tenant isolation.
  • Code inside the attacker-controlled sandbox reads the forwarded session token from process memory and exfiltrates it to an external server.
  • The attacker replays the stolen token to gain full control of the victim's Writer account — accessing private chats, documents, agent configs, private models, connectors, and LLM credentials.
  • Guardrail bypass was straightforward: instead of pasting malicious code inline, the agent was told to fetch and run a remote script — the input-side filter saw a benign "download and run" request.
  • Writer patched the issue by preventing session cookies from being forwarded into sandbox previews and moving them to an isolated origin.

Why It Matters

This is a textbook failure of the shared responsibility model in multi-tenant AI platforms. The guardrails checked the instruction, not the runtime behavior. Enterprise AI platforms that allow user-defined code execution in managed sandboxes must treat session isolation as a transport-layer problem, not an application-layer one. Any platform with agent preview/sandbox features faces the same architectural risk.

What To Do

  • If you use Writer AI: confirm you're on the patched version and review access logs for suspicious agent preview activity.
  • For any multi-tenant AI platform: ensure session cookies are never forwarded into user-controlled sandbox environments.
  • Move sandbox previews to isolated origins with no cookie-sharing path back to the main application.
  • Audit agent code execution guardrails — input-side filtering is insufficient; enforce runtime behavior constraints.
  • Implement cross-tenant isolation testing as part of your AI platform security review.
