Varonis — 'Rogue Agent' GCP Dialogflow Code Block RCE, Conversation Exfil

2026-07-08 Security by al-ice.ai Editorial

AI relevance: Google Cloud Dialogflow CX is a production AI agent platform used by enterprises to build customer-facing chatbots; this vulnerability lets attackers hijack the agent's execution environment and exfiltrate all conversation data.

  • Varonis Threat Labs disclosed "Rogue Agent," a critical vulnerability in Google Cloud Platform's Dialogflow CX that affects all agents using Playbook Code Blocks within the same GCP project
  • The flaw abuses Dialogflow's Code Blocks feature, which lets developers embed custom Python logic to process user input and call external APIs
  • All agents in a GCP project share the same Cloud Run execution environment, and researchers found that code_execution_env.py — the file that runs Code Block logic via Python's exec() function — was writable and lacked code restrictions
  • By overwriting this file, an attacker gained access to shared session variables like conversation history and could hijack every agent's execution scope on the project
  • Only the dialogflow.playbooks.update permission was needed to configure Code Blocks and execute arbitrary Python — this permission can be scoped to a single agent
  • Attackers could exfiltrate conversation data to external servers, impersonate the agent's legitimate responses using internal functions like respond(), and inject phishing prompts disguised as reauthentication requests
  • Once the malicious code was persisted, attackers could restore the original-looking configuration in the console, making the compromise invisible in Cloud Logging
  • Two compounding issues amplified the risk: VPC-SC bypass (Cloud Run's unrestricted outbound internet access let attackers turn the execution environment into a covert data-exfiltration proxy) and IMDS credential leakage (exposure of the Instance Metadata Service allowed retrieval of access tokens tied to a Google-managed service account)
  • Google shipped an initial fix in April 2026 and fully resolved the issue by June 2026; no known in-the-wild exploitation before the patch

Why it matters

This is the third major AI-platform vulnerability Varonis has disclosed, following Reprompt in Microsoft Copilot Personal and SearchLeak in Microsoft Copilot Enterprise (CVE-2026-42824). The pattern is clear: AI agent platforms are expanding rapidly (roughly 80% of Fortune 500 companies now actively use AI agents), but security controls haven't kept pace. The shared execution environment model in Dialogflow CX created a single point of failure — one compromised agent could poison all agents in the project. The VPC-SC bypass is particularly concerning because it defeats a core Google Cloud security control, turning the AI agent into a data exfiltration proxy even when network isolation is enforced.

What to do

  • Enable DATA_WRITE audit logs for the Dialogflow API and review past playbook update events for anomalies
  • Correlate suspicious updates with rare API access, unusual IP addresses, or atypical access times
  • Query Cloud Logging for failed requests and inspect protoPayload.status.message for exceptions tied to malicious Code Block logic
  • Manually review each agent's Playbooks in the Dialogflow CX console to confirm only whitelisted Code Blocks are configured
  • If you used Dialogflow CX with Playbook Code Blocks before the April 2026 patch, assume potential compromise and rotate any credentials that may have been exposed
